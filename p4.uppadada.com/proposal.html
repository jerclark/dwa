Big Meals, No Whammies

Abstract

Big Meals, No Whammies is a web application that will help people plan meals and shopping lists for themselves or their families. This application is inspired by my family's recent usage of the meal plans and shopping lists available from 100daysofrealfood.com.

Usage Overview

Using Big Meals, No Whammies is relatively straigtforward. The basic flow is:
  • Create an account or Login to begin
  • Add recipes (with instructions and ingredient lists) to your Recipe Bank
  • Use the "Meal-o-Matic" to generate a "Big Board" of meals. The application will generate a meal plan, suggesting a breakfast, lunch, snack and dinner option for every day of the date range of the meal plan
  • Adjust your meal plan by rearranging the suggested options, or by using alternatives from your recipe bank
  • Use the auto generated "shopping list" to buy the necessary goods for the weeks meals

UI/Features

Main Screen

The main screen will contain most of the UI necessary to use the site, including: The Recipe Bank - users can add/edit/remove their own recipes, and see other user's shared recipes The Big Board - User's can choose a date range for which to plan meals, and view a grid of the meal plan. The user can also click the "View Shopping List" button to view the shopping list associated with a meal plan.

Recipe Bank

This is where the user will Manage and Browse recipes. The user can:
  • Add recipes, and Edit/Remove recipes they own
  • Enter ingredients, steps and metadata about recipes (metadata: type of meal, healthy/junk, cost, ease, time, sharing preference, pictures, custom tags?)
  • Add Metadata about ingredients (cost, season, availablility, images)
  • Browse other users shared recipes
  • Search for recipes (By Name, Ingredient, user, other metadata>)

The Big Board

This is where the user will create meal plans
  • User can set a date range for which to make a meal plan (I will probably set some limit here)
  • User can do a random "Auto-Generate" from the recipe bank
  • User can do a guided "Auto-Generate" from the recipe bank by entering some "parameters" to the generation process (possibilities: favor "healthy" recipes, allow higher tolerance for "junk" food, keep overall ingredient list below some threshold, favor in season recipes, etc.)
  • Manually drag/drop recipes from the recipe bank
  • User can rearrange recipes in the grid by drag/drop
  • User can click the "View Big List" button to view the Big List (shopping list)

Nice to haves

  • User can export plan to CalDAV services

The Big List

This is the shopping list for the meal plan
  • The user can e-mail the list (for consumption on a smartphone, etc)
  • The user can print the list
  • The user can view the list on a smartphone
  • The user can add or edit an existing price for an ingredient
  • If the user hasn't added a price, they can view an average of all other users price entries for that ingredient
  • The user can check items as they are purchased

Nice to haves

  • User can export plan to some list/task (like ta-da-list, or some other task/list manager)

Other Inspiration/Examples

As stated earlier, I've decided to do this based on my family's recent foray into meal planning. There are MANY meal planners out there, some of them are pay services and others are free. I'm not sure I will put any of them out of business with my efforts, but I do think it's a good way to apply technology to something I'm interested in, go through the rubrick of P4, and try to get creative an differentiate a little bit.
  • 100daysofrealfood.com - This is a blog site in which the blogger posts recipes and other food tips, all around the home-baked concept of "real" (i.e. non-processed) food. She shares meal plans and shopping lists, and I have found them useful. But they are static PDFs - it was my desire to make them more dynamic that lead me to want to make this project.
  • Food on the table - This is a pay site. Looks pretty nice, and appears to have integration with grocery stores and specials. It costs money, so I didn't pay to sign up.
  • Ziplist.com - This is a free site, and it's pretty comprehensive in terms of sharing recipes, integration with stores/specials and generation of shopping lists. It also integrates with different calendar technologies, which is pretty cool. I'm not crazy about the planning UI.
  • EatingWell.com Menu Planner - This is also a free site - its more focused on calories count/weight management, as the first question you're asked is about BMI. It's obviously an add-on to a magazine with other motivations, but I do like the planning UI a little more than zip list:
